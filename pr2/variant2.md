# Індивідуальне завдання (Варіант 2)
**Тема:** Дослідження впливу використання mmap() замість malloc().

## 1. Постановка задачі
Потрібно з'ясувати, що працює швидше: стандартна функція виділення пам'яті `malloc()` чи прямий системний виклик `mmap()`.

Для цього я написав дві програми (`malloc_t.c` та `nmap_t.c`), які роблять одне й те саме:
1. У циклі (10 000 ітерацій) виділяють блок пам'яті розміром 1 МБ.
2. Записують туди дані (щоб пам'ять фізично виділилася ОС).
3. Звільняють пам'ять.

**Результат для mmap (`./tnmap`):**
```bash
$ time ./tnmap

real	0m1.481s
user	0m0.350s
sys	0m1.130s
```
**Результат для malloc (./t_malloc):**

```bash

$ time ./t_malloc

real	0m0.003s
user	0m0.001s
sys	0m0.002s
```
## Висновки та аналіз
**Експеримент показав, що `malloc` працює у сотні разів швидше за mmap.
Високе навантаження на систему (sys): У результатах tnmap видно, що час sys складає 1.130с. Це означає, що процесор витратив цей час на виконання системних викликів (перемикання в режим ядра). `mmap` щоразу звертається до ядра ОС, щоб отримати сторінки пам'яті, що є "дорогою" операцією.
Оптимізація `malloc`: Функція `malloc` працює у просторі користувача (user space). Вона один раз запитує у системи великий шматок пам'яті, а потім сама нарізає його на менші частини для програми. Це дозволяє уникнути постійних звернень до ядра.
Використання `mmap` для частого виділення пам'яті є неефективним через накладні витрати на системні виклики. Його варто використовувати лише для специфічних задач (наприклад, робота з великими файлами або IPC), а для звичайних потреб краще підходить `malloc`.**
